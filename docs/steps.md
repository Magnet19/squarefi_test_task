# Пошаговый план реализации (Phases)

Разработка проекта разбита на четыре основных этапа, чтобы гарантировать правильную интеграцию Next.js App Router, защищенных Server Actions и Single Refresh паттерна. Детализация по конкретным задачам находится в файле `tasks.json`.

## Фаза 1: Инициализация и Базовая Архитектура (Завершена)

Создание фундамента проекта, настройка строгой типизации и изоляции сессий.

- Инициализация пустого Next.js 16.1.6 проекта (App Router) со всеми необходимыми пакетами (`shadcn/ui`, `zod`, `sonner`, `server-only`).
- Написание типов для работы с DummyJSON (Products, User, AuthResponse).
- Реализация `getValidAccessToken` и `refreshAccessToken` с In-Memory Promise Dictionary для Single Refresh паттерна.
- Реализация типобезопасной обертки `apiFetch` (серверный fetch).
- Настройка `proxy.ts` (замена middleware) для защиты маршрутов и Server Actions авторизации/выхода (`loginAction`, `logoutAction`).

## Фаза 2: Аутентификация и UI

Создание визуальной части для входа в систему и интеграция с готовыми Server Actions.

- Создание клиентского компонента `LoginForm` (с Zod-валидатором).
- Использование Server Action `loginAction` в форме через `useActionState` (или обычный submit).
- Отображение ошибок (Toast, inline errors) используя паттерн `Result<T, E>`.
- Сборка страницы-заглушки для роута `/login`.

## Фаза 3: Dashboard и Список продуктов

Рендеринг защищенных данных полностью на стороне сервера с использованием `apiFetch`.

- Создание Server Component на маршруте `/dashboard/page.tsx`.
- Получение продуктов с пагинацией (чтение `searchParams` для определения номера страницы `?page=X`).
- Создание UI-компонента карточки продукта (`ProductCard`).
- Создание клиентского компонента пагинации (`Pagination`), который обновляет URL без сброса стейта приложения (используя `<Link>` или `useRouter`).

## Фаза 4: Взаимодействие с корзиной

Завершение функционала магазина через мутации на стороне сервера (Server Actions), без использования клиентского стейта (Zustand/Redux).

- Написание валидируемого Server Action для добавления товара в корзину (`addToCartAction`, интеграция с `/carts/add` DummyJSON).
- Добавление клиентского компонента `AddToCartButton`, который вызывает этот Action при клике.
- Реализация `revalidatePath('/dashboard')` или `revalidateTag` при успешном добавлении в корзину, чтобы интерфейс актуализировался новыми данными (например, для счетчика корзины).
- Установка и интеграция `sonner` для тост-уведомлений об успешном добавлении товара.
